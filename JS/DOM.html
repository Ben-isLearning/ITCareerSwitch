Document Object Model (DOM)

DOM

JS Doesn't exist in a vacuum. JS operates alongside HTML web pages.
Understanding the DOM is a nessaccary step of understanding JS.
The D-O-M;
Document - The webage its self
Object - The objects that are created based on the HTML structrue
Model - A consistent approach to the various objects

In short ~ The DOM is a schematic of a web page,
the objects avaliable, and the methods of accessing them.

The DOM defines;
each different HTML Element as an Object
The properties of all HTML elements
The methods to access all HTML elements
The events for all HTML elements
(Responding to Clicks / Mousovers / Etc)

DOM Properties are values that we can set or change
(Ex; Changing the Content of an HTML element)
Whereas
DOM Methods are actions that we can take on HTML elements
(Ex; Adding or Deleting an HTML element)

Changing DOM

We are able to modify elements entirely, or in part by via attributes.
Changing uses the set method; to give values.
Rather than the get method, which retreives values.
Ex;
document.getElementById("openingParagraph").innerHTML;
Will display the current value of the innerHTML of "openingParagraph"

Whereas setting...
document.getElementById("openingParagraph").innerHTML = "Text Changed Successfully"

Ex;
document.getElementById("dogPhoto").getAttribte("align");
Will return the value of the align property of the particular element

Whereas set...
document.getElementById("dogPhoto").setAttribute("align", "center");
Will set the value of the align property as given.
This method requires ("Name", "Value").
This method be used to change Image sources after a given period of time,
effectively creating a slideshow.

CSS properties can also be amended with this method,
However it is bad practice to do so. Static styling should be handled externally within the .css
But we are able to program responsiveness in this manner.
Font-family: becomes fontFamily, etc, etc
Ex;
doucment.getElementById("p2").style.color = "blue";
This can also be acheived by
setAttribute("style", "color:blue");

This example is merely an abstract proof of concept.

Responding to Events

JS is meant to Control the behaviour of the webpage and its interactivity.
Our code should be able to respond to actions that are taken on the page!
This is acheived through Events and Event Listeners...
We do not cause Events; Scrolling, Resizing, Hovering, Clicking, Changing Focus, etc.
We chose what events we want to listen for, and react to.
And we chose an appropreate reaction for each specified event.

There are 3 ways that we can respond to events;
1) Write JS code directly into HTML (Bad Practice - Don't Mix Languages)
2) Specify element and event we want to handle, then write a function (Best Practice!)
3) use the addEventListener() function (Best Practice!)

1) Mix HTML + JS (Bad Practice);
<button onclick='alert("Hello world!");'>Do Something Crazy</button>

'Responding' to Events - In due time!

There are times when we want behaviour on a page to be controlled automatically by time
for instance, after 15s a subscription popup occurs offering 10% off the order
While these aren't technically events, they operate very similarly.
Two functions will provide time-based responsiveness
setTimeout() and setInterval()
Both of these functions take (functionToBePerformed, Time in Milliseconds)

Ex;
Set timer... for Function to be performed ONCE, IN... 5000 ms! (5s)
function myFunction() {
alert("This is a message")
}
setTimeout(myFunction, 5000);

Set timer... for function to be performed EVERY... 5000 ms! (5s)
function anotherFunction() {
alert ("See you in 5 seconds!")
}
setInverval(anotherFunction, 5000);
