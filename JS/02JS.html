Naming Conventions / Best Practices

To create reliable, consistent code that doesn't make your eyes burn
Best Practice = Informative, Descriptive name
submitButton rather than let1
infoTextBox rather than myVariable

let speedLimit = 70;
Rather than
let number1 = 70;

Perhaps match Class or ID of HTML Elements with Variable names of JS
Majority of JS uses camel notation;
camelsHaveOneSmallHumpThenManyLargeHumps

Some instances use the Pascal notation,
only in specific circumstances (Covered Later);
PascalLikesGoingHardFromTheBeginning.

Spacing: Always use spaces around operators and after commas
let one = 1;
Rather than let one=1;


Literal Values

Number Literals: 4 | 4.5
(It's literaly a Number)

String Literals: "one" | "5" | 'abc101'
(Its literally a String of Characters "" or '' is fine)

Array Literals: ["1", "2", "3",]
(Its Literally a BOX full of values)

Boolean Literals: true | false
(Its Literally true or false (1 or 0))

Object Literals: {Property(Key):Value}
(Literally CURLYBRACES filled with Key:Value pairs

Variables: let x = 6;
(Literally a declaration that x is equal to 6)


Operators

Assignment Operator

=
Used to Assign value.
( let x = 12; )

Arithemetic Operators

+ | - | * | /
Used to Compute values.

(let x = a + b;)
(let y = a - b;)
(let alpha = x*y;)
(let Beta = x/y;)

Comparison Operators

Used to compare values.
== | === | != | < | > |
if ( 1 == '1') ~ Loose Equality - Looks at Value (True)
if ( 1 === '1') ~ Strict Equality - Looks at Value + Type (False)
if ( x != 5 ) ~ NOT equal - Determines if 1st Value is NOT equal to 2nd Value (True)
if ( x < 6) ~ LESSTHAN - Determines If 1st value is SMALLER than 2nd value
if ( x > 6) ~ GREATERTHAN - Determines if 1st value is BIGGER than 2nd value


Expressions

An expression is a combination of Values Variables and Operators
which computes to an evaluation (A Result)

Expressions:
15 * 10;
myValue+10;
"John" + "" + "Doe";


Keywords

** Keywords cannot be used as variable names**
** Give Variables Meaningfull names **

let - Declares a Variable
for - Marks a block of code to be executed for as long as a condition is true
if ... else - Marks a block of code to be executed based on a condition
return - Exits a function
break - Terminates a switch or loop
function - declares a function


Comments

Create comments in JS by typing // before
If its a multiline Comment, Begin with /* and end with */
Add Descriptive comment before code, not after.
States Purpose of code - Public Comments
