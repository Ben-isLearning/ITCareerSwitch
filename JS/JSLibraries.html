Common Issue; Scripts Running before page is loaded

effectively preventing the scripts from accessing the HTML elements on the page.

jQuery overcomes this with the ready() function, that waits for all HTML elements to load
before executing any code.

Essentially, just place the code inside the ready() function.
The Below code will wait until the webpage has loaded to perform the Alert

$(document).ready(function() {
alert {"Hello World"};
});


JavaScript vs jQuery

function myFunction () {
let object = document.getElementById("alpha");
object.innerHTML = "Hello jQuery";
}

against...

function myFunction() {
$("#alpha").html("Hello jQuery");
}

And to ensure it doesn't Invoke before DOM is finished...

$(document).ready(myFunction); jQuery is much more efficient!

jQuery Methods

Jquery has powerful methods for changing and manipulating HTML content.
(Both content and styles)
Various DOM methods exist within jQuery...

Primary Methods:
text() - sets or returns text content of the selected element
html() - sets or returns all content of the selected element (including HTML markup)
val() - sets or returns values in form fields

Ex 1 + 2: text() + html()
The below codeblock has 2 functions, nested inside of a 3rd ready function
The .text() shows the Text
the .html() shows all of the content including the html

$(document).ready(function() {

$("#btn1").click(function() {
alert("Text: " + $("#test").TEXT());
});

$("#btn2").click(function() {
alert("HTML: " + $("#test").HTML());
});
});
