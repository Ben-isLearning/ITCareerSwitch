Positioning Within the Grid ~ grid-xxx-xxx;

To control the position of the element within the grid, there are four elements.
grid-column-start:
grid-column-end:
grid-row-start:
grid-row-end:

The potential values for all of these properties are;
1 (numbered grid line)
span 3 (item will span across the provided number of tracks)
span "name" (item will span across untill it hits the next line with the name provided)
auto (automatically do it)

Example;
.item-a {
grid-column-start: 2;
grid-column-end: 5;
grid-row-start: row1-start;
grid-row-end: row2-end;
}

Example 2;
.item-b {
grid-column-start: 1;
grid-column-end: span column4-start;
grid-row-start: 2;
grid-row-end: span 2;
}

Shorthand for Positioning - grid-xxx:

The shorthand properties for this is grid-column: and grid-row:
ofcourse, these properties take 2 values, the starting point and the ending point.
As such, the previous examples become...


Example;
.item-a {
grid-columm: 2 / 5;
grid-row: row1-start / row2-end;
}

Example 2;
.item-b {
grid-column: 1 / column4-start;
grid-row: 2 / span 2;
}

The grid-template-areas: property allows us to create an 'area' of the grid
by referencing the individual names of the grid squares.

Much like Javascript Functions, the grid-template-areas should be defined, with "grid-template-areas:" then called,
Called with the grid-area: property, supplying the name defined prior.

Painting the Walls! ~ grid-area:

the grid-area: property references the names created within the grid-template-areas: property
OR it can be used as shorthand for both grid-column: and grid-row: properties.
The Shorhand property requires 4 values; Row start / Row end / Col start / Col end.

If using name;
.item1 {
grid-area: specifiedName
}

If using as shorthand;
item2 {
grid-area: 1 / column4-start / 2 / span 2;
}


Moving single cell contents within Main Axis ~ justify-self:

justify-self: property takes 1 value,
And behaves much the same as justify-content: and justify-items:... But on a SINGLE CELL.
justify-self: start | end | center | stretch;


Moving single cell contents within Cross Axis ~ align-self:

align-self: property takes 1 value,
and behaves much the same as align-content: and align-items:... but on a SINGLE CELL
align-self: start | end | center | stretch;


Shorthand for single cell Both Axis ~ place-self;

place-self: property takes 2 values,
and behaves much like place-items: and place-content:
place-self: align-self Value justify-self Value;


Sizing Conventions

Use fr - Fractional Units.
for example; if the entire space avaliable was cut up into equal fractions,
Allocate the space accordingly...
Ex;
grid-template-columns: 1fr 3fr; ~ Would divide space 25% / 75% between the two grid columns.


Sizing Keywords

In addition to px, rem, %, we also have sizing keywords...
min-content:
Set a value that is the smallest you want the content box to be
max-content:
set a value that is the largest you want the content box to be
auto:
alot like fr... but loses out to fr..
fit-content:
use the space avaliable, but never less than min-content and never more than max-content


Grid Specific Functions

repeat()
The repeat(X, Y) function literally repeats an action X times, at Y size;

grid-template-columns: repeat(6, 1fr);
This yeilds 6 columns - each with 1fr of space.

minmax()
The minmax() function defines a size range greater than or equal to min, and less than or equal to max.
The difference turns:
grid-template-columns: repeat(3, 1fr);
into
grid-template-columns: repeat(3, minmax(10px, 1fr));
The minmax() ensures that the columns do not go below 10px, nor do they exceed 1fr (in this example)

auto-fill the auto-fill keyword will automatically fill as many possible columns on the row, even if they are empty.
Auto-fill ~ Fill this row full o' columns, Don't care whats in em'!

auto-fit the auto-fit keyword will automatically fit the given amount of columns into the row.
auto-fit ~ Expand the full columns to fit this entire Row!
