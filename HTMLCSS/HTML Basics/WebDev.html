Development Process

Series of phases to complete the project timeframe
SDLC (Software Devlopment Life Cycle)
Concept
Analysis
Design
Testing
Implementation


Conceptualization
Taking a concept and evaluating if the concept will succeed.
+ Define purpose to create foundations for development


Analysis
Determine reach / functionality / information topic
Determine information topic - What is the purpose of the website? Why are we doing it? why are we doing it this way?
Determine functionality requirements - What kind of experience will our users expect? complicated? simple?
Determine environmental requirements - what hardware is needed + how much bandwidth do we have?
Determine content requirements - what do we want to convey? what topics are currently in the "bubble"?
Review the competition's websites - what is good / bad
Estimate costs - what is it going to take to acheive this project?
Can we ship the project in stages? Alpha beta etc?


Design
Prepare and plan concepts, purpose and functionality of site.
Chose site organisation - Pick a structrue - Herarchical vs Linear
Design a prototype - give owners a prief look at what the site will do
Design a page layout - sketches / wireframes to determine the layout
Document each page - what does the page do spefically? (ie about page) keep track of changes.


Production
Bringing small peices together
what platform are we devloping?
authoring tool? WordPress / Google / Hand-code?
Organize the site, how do we store elements? what is the directory structure? naming schemes?


Develop and Test
Break the functionality down into small peices and test.
Test on different browsers - chrome / opera / firefox / safari
test on different devices - desktop / mobile / layout
test different screen resolutions -
test bandwidth - G2 G3 etc?
test outside host location -
test test test till it breaks
create physical document for testing requirements.
Automate testing - scripts / bots - Bot reporting
Accesibility Testing - meeting standards, zoom? color blind? voice narative? operable, understandable + robust
Test using more than one methods.- "Try to break this?"
Usability testing - Intuitive design / ease of learning / efficiency of use / memorability / error frequency + severity / subjective satisfaction
Launch - Get it out there!
Maintenance - Updates Software / Hardare
Evaluation - Does the final design/implementation meet all the requirements?

Register domain before someone else does. Keep registration services private.
Physical host? Virtual host?
