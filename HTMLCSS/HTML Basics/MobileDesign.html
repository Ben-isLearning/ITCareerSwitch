CSS Media Queries

Media queries literally do just that; CSS Questions the Media perameters
to establish screen size, resolution, orientation, browser type, etc

Media Type;
@media all - (All devices)
@media print - (Printing the webpage only)
@media screen - (Computer Screens only)
@media speech - (For speech synthesizers)
@media only "" - (Hides media query from older browers)

Ex;
<link href="something.css" rel="stylesheet" media="only screen and (min-width:768px) and (max=width:1024px)">

(The (min-width:768px) Act as Boolean Conditions just like JS - If false, it wont run)
something.css is only referenced when;
Using a new browser,
Viewing from a computer screen
That is bigger than 768px
And small then 1024px


Flexible Images

Flexible images with CSS.
To ensure that images are sufficient in both desktop and mobile useage.
Remove height and width attributes from image properties
Set max-width:100%;
Set height:auto; (To maintain aspect ratio)

A static image will retain its hard-coded size regardless
400px on desktop = small | 400px on mobile = huge!

static image {
height:400px;
width: 400px;
}

A fluid image has an ambiguous size, adapting to its surroundings.
100% of width on desktop = huge! | 100% of width on mobile = huge!

fluid image {
height:auto;
width: 100%;
}


Picture Element

The <picture> element allows developers to serve different images
based on specific criteria. (Bigger screen gets image A, smaller screen gets image B)
srcset="" = Default image source

<picture>
<source srcset="/media/cc0-images/surfer-240-200.jpg"
media="(min-width: 800px)">
<img src="/media/cc0-images/painted-hand-298-332.jpg" alt="" />
</picture>

This picture tag sets the source (srcset="") to a default image
Then changes the source (src) when a condition is true (min-width: 800px)
Effectively serving two different images depending on the screen size.
