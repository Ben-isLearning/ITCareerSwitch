CSS box-sizing

Ideally, the entire project needs to be one or the other
By default box-sizing: content-box; follows this formula:

width(300px) + padding + border = actual width
height(100px) + padding + border = actual height

But box-sizing: border-box; changes the formula:

width(300px) = actual width
height(100px) = actual height

Positioning With CSS;

position: static; (Default value - Rendered in normal flow)
position: fixed; (Keeps element or object Fixed in location - Even when scrolled)
position: relative; (positions element relative to where it would be in normal flow)
position: absolute; (position element to exact location on page, regardless of flow)

Structrued Elements

Section | Article | Aside | Time
for more semantic elements, visit W3Schools

Section

"A section is a thematic grouping of content, typically with a heading."
the <section> element defines a section in a document.
Sections could be; Chapters | Introductions | News Items | Contact Information | etc
Anything that is a group of content in and of itself.
Example: a single Section full of articles

Article

The <article> element specifies independent, self-contained content.
An article should make sents on its own, and it should be possible to
distribute it independantly form the rest of the website.
Articles could be; Blog Posts | Tweets | User Comments | etc
Anything that could be a stand-alone item

Aside

The <aside> element defines some content aside from the content it is placed in (like a sidebar).
the <aside> element should be indirectly related to the surrounding content
For example, an article explaining the importance of an obscure ???Rare Plant???
with an aside that defines and displays the ???Rare Plant???'s rarity

Or.., an article explaining the death of Steve Irwin,
with various Asides about his family members, professions, Stingrays, etc
