Building the Walls ~ grid-template-...

The two properties grid-template-columns: and grid-template-rows: define the
amount of rows and columns, their sizes, and their names.
These properties take 2 values;
1st) line-name ~ which can be anything we chose, inside [ ] .
2nd) track-size ~ which can be a length (100px), percentage (30%), fraction of free space (1fr | 4fr)...
or even the repeat function (which takes (amount, space)) [Ex: repeat(5,1fr)]

Set some areas ~ grid-template-areas:

The grid-template-areas: property allows us to create an 'area' of the grid
by referencing the individual names of the grid squares.
grid-area-name: can take three values; {Name of Area} | . (Meaning Empty Cell) | none (Meaning no areas are defined)
Much like Javascript Functions, the grid-template-areas should be defined, with "grid-template-areas:" then called,
Called with the grid-area: property, supplying the name defined prior.

Make some space! ~ gap:

The column-gap: and row-gap: properties define the space between grid cells.
Both properties only take one value, the size of the gap.
The shorthand of these properties is simply gap:
the gap: property takes 2 values; row gap value, column gap value.
row-gap: 10px; column-gap: 15px; AND gap: 10px 15px; do the same thing.

Shorthand for align-items: + justify-items: === place-items:

the place-items: property is shorthand for both align-items: and justify-items:
the place-items: property usually needs 2 values: align-items value, justify-items value;
however if its the same value, 1 will work.
Really good if you wanna slap your content in the center of its grid cells
place-items: center;


Move entire Grid along Cross Axis = align-content:

If the overall size of the Grid is less than the overall size of its container
we can set the alignment of the entire grid within its container;
the align-content: property behaves similarly to align-items: above.
but the difference is its moving ALL CONTENT, rather than SINGLE ITEM

What if there are more cells than Grid? ~ grid-auto-xxx:

The grid-auto-rows: and grid-auto-columns: properties enable us to specify the size of
any automatically generated grid tracks.
Realistically if you've specified a 3 x 3 grid, but you've got 12 content elements, thats when you need grid-auto!
both grid-auto-rows: and grid-auto-columns: take 1 perameter, the size of the grid you want.
The single value can be lengths, a percentage or a fraction of the free space (fr)

What about the flow of the automatically placed cells? ~ gid-auto-flow:

When automatic grid cells are placed in, they need to be instructed what way to flow too.
the auto-placement algorithm kicks in, set its direction by using the grid-auto-flow: property.
the grid-auto-flow: property only takes 1 value, but it can take;
row; | column; | row dense; | column dense;

Here the grid-auto-flow: column; has been ruled. - Fun!
