Forms

Forms are everywhere

Any instance of a user inputting data on a website is acheived through use of a form.
Inputting your name into an account? input!
signing in with an email address? input!
sending an email? input!
literally anything you're entering. Input!
Any time you hit submit button after entering data, Its a form!

Forms consist of HTML code on the front, and server-side data processing on the back.


Data != Information

Data is ambiguous, raw and truly meaningless.
2 or more peices of data = Information.
Information is key!

For instance;
if we take "red", it doesn't actually mean anything,
it could mean a color, it could mean a name, it could mean something else.

Then;
if we take "red" + "car", two peices of data concatinate into "red car"
which gives us information that is usable.


Form element ~ <form>

The form element has of 5 attributes;
action - (the url or filename/path to the server)
autocomplete - (???)
id - Allows CSS Manipulation
method - The fuction of the form (get / set / etc ))
name - (RegForm / OrderForm / Meaningful name)

syntax:
<form name="orderForm" method="post" id="orderForm" action="processData.php">
Best practice Name ==== Id

Inside the form element, we require both the input element,
(That sets the type and allows the input)
and the label element,
(That shows the user what the input is for)
lable.for ==== input.name


Form Controls / <input> element

<input> element attributes;
type="" - Sets type of input field
name="" - Sets name of input
id="" - Allows CSS Manipulation
value="" - Sets default value
accesskey="" - (Hotkey) Allows form-field to be selected via hotkey
tabindex="" - Order of advancement (Stop TAB going everywhere (Starts@1))
checked - Boolean value - Sets an input field to be checked by default
disabled - Boolean Value - disables an input field entirely (no user / server)
readonly - Boolean Value - Restricts user input - Still active
autofocus - Boolean Value - Sets form field to be focused by default
required - Boolean Value - Requires user input (Basic validation)

Submit
Used to submit or begin executing the method=""
<input type="submit"> The <input> element places a user control on the form

Reset
Used to clear the form - effectively reset all feilds.
<input type="reset">

Check Box
Used to select one or more options - can be unchecked
<input type="checkbox">

Radio Button
Used to select one choice out of multiple options.
(All options need to have the same name="" attribute)
<input type="radio">

Hidden input
Used to store information that is not visible in the viewport
THIS DATA CAN STILL BE ACCESSED VIA DEV-TOOLS
<type input="hidden">

Password Input
Same as text box, but the characters are obscured.
<type input="password">
