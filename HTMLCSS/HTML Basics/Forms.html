Forms

Forms are everywhere

Any instance of a user inputting data on a website is acheived through use of a form.
Inputting your name into an account? input!
signing in with an email address? input!
sending an email? input!
literally anything you're entering. Input!
Any time you hit submit button after entering data, Its a form!

Forms consist of HTML code on the front, and server-side data processing on the back.


Data != Information

Data is ambiguous, raw and truly meaningless.
2 or more peices of data = Information.
Information is key!

For instance;
if we take "red", it doesn't actually mean anything,
it could mean a color, it could mean a name, it could mean something else.

Then;
if we take "red" + "car", two peices of data concatinate into "red car"
which gives us information that is usable.

Form Controls / <input> element

<input> element attributes;
type="" - Sets type of input field
name="" - Sets name of input
id="" - Allows CSS Manipulation
value="" - Sets default value
placeholder="" - Acts like value="", but text disappears when selected.
accesskey="" - (Hotkey) Allows form-field to be selected via hotkey
wrap="" - configures line breaks within text fields.
(Soft = linebreak @ border / Hard = linebreak @ row="" value)
tabindex="" - Order of advancement (Stop TAB going everywhere (Starts@1))
checked - Boolean value - Sets an input field to be checked by default
disabled - Boolean Value - disables an input field entirely (no user / server)
multiple - Boolean Value - Allows user to select more than one option
readonly - Boolean Value - Restricts user input - Still active
autofocus - Boolean Value - Sets form field to be focused by default
required - Boolean Value - Requires user input (Basic validation)
maxlength="" - Validation - Sets maximum character length allowance
minlength="" - Validation - Sets minimum character length allowance
max="" - Validation - Sets maximum value for numeric fields.
min="" - Validation - Sets minimum value for numeric fields.
list="" - Operates like for===Id, but for multiples list===id

Form Controls Accessability

If we just left an empty input field on a webpage, a user wouldn't
inately know what data the webpage is asking for.
As such, we need to include accessiblity options to our forms.

Label element
the label element is used to provide associated text to go along with
an input field.
Option 1
Link the input and the label elements by matching
the Id="" of the input
and the for="" of the label

Standard Ex:
<label for="nameInput"> Name here </label>
<input type="text" id="nameInput">



For increased accessability: put everything in the label,
then the padding can be clicked to access the input element.



Server-side Processing

Consists of a request of some sort from the web browser
the request is processed (Get / Set / Etc)
and the server returns requested information

Much like PersonA asks "whats the time?"
PersonB responds "Mid day"
then PersonA knows what time time is.

Examples of server-side procesing languages
php
ruby on rails

Server-side processing is
(Functionality) set up by the "method" attribute within the <form> element.
(File Path) triggered by the "action" attribute within the <form> element.

Method attribute defines how to send form-data.
Action attribute defines where to send form-data

Example;
<form method="post" action="www.something.net/scripts/Add.php">
Will Post data to the add.php function
